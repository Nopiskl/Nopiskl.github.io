1. CSI2 设备先 probe

   - 占用 PA6-PA11

2. MIPI1 设备后 probe
   - 尝试应用 `mipib_pins_a`

### 3. **驱动代码层面的矛盾**

```

虽然代码中 MIPI 模式不请求 pinctrl，但设备树中的 `pinctrl-0` 仍会在 probe 时被应用，导致冲突。

### 4. **为什么会出现这个错误**

从错误日志看：

```

- `5810200.mipi` 是 MIPI1 的设备地址（0x05810200）
- CSI2 已占用 PA6
- MIPI1 无法再申请同一引脚

### 5. **初始化失败链**

1. MIPI pinctrl 请求失败 → `__mcsi_pin_config` 返回错误
2. MIPI 初始化失败 → 无法配置 MIPI 硬件
3. Sensor 初始化失败 → `kw33000_init_regs` 失败（因为 MIPI 链路未建立）
4. Stream on 失败 → `VIDIOC_STREAMON` 失败
