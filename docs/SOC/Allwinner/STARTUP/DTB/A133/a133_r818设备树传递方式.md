# A133/R818 默认启动链路中的 DTB / FDT 关系（修订版）

## 最终结论（推荐直接引用）

如果要给出一段最严谨、最不容易引起误解的总结，推荐直接使用下面这段：

> **A133/R818 默认启动链路并不是“从上电开始就完全只使用 kernel 那份 DT”。U-Boot 早期仍然需要先依赖自己的 control FDT 完成启动阶段的设备描述、参数读取和启动决策。只是与 A733 不同，A133/R818 默认没有开启 `CONFIG_OF_SEPARATE + replace_fdt` 这类后半程显式切换机制，且 `boot.img` 默认也不是第二 DTB 来源，因此通常不会在临启动内核前再切到另一份独立 Linux DTB，而是在当前 working FDT 基础上修补后直接交给 kernel。**

再压缩成一句话就是：

> **A133/R818 默认不是“完全只用 kernel DT”，而是“没有 A733 那种后半程显式双树切换”。**
